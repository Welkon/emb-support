# TQ322

ID: `components/tq322`

## 用途

这个摘要用于支撑 `TQ322` 相关项目中的器件识别、接口建模和 MCU 侧读帧实现约束判断。

## 共享结论

- `TQ322` 不应默认当成普通高低电平 `PIR` 模块
- MCU 侧应按单线数字帧接口理解 `TQ322` 的 `S` 脚，而不是按简单电平输入处理
- 数据从 `MSB` 开始读取
- 总帧长为 `19bit`
- 帧格式为：`D18=1`、`D17=0`、`D16..D1=16bit signed data`、`D0=0`

## 关键时序约束

- 被动或中断模式下，`S` 脚出现约 `0.0625ms` 高电平后，MCU 检测到 ready，再等待至少 `100ns` 才开始读时序
- 主动或强制读取模式下，MCU 可主动把 `S` 脚拉高至少 `0.0625ms`
- 单 bit 读取时，MCU 先把 `S` 脚拉低，低电平宽度至少 `200ns`
- 随后对 `S` 脚产生上升沿，并保持高电平至少 `200ns`
- 传感器接管后，MCU 应在固定相对时刻采样，不能让每一位走不同指令路径
- 第 `19bit` 读完后，MCU 需要把 `S` 脚强制拉低再立即释放，或保持低电平至少 `64` 个系统时钟周期，用于触发器件内部数据锁存更新
- 如果中途放弃并希望更新内部数据，`S` 脚低电平必须超过 `1` 个系统时钟周期；若中途强拉高，则会终止读取但不更新数据

## MCU 侧直接约束

- 需要稳定、可控的短延时能力，才能满足 `100ns` / `200ns` 级时序要求
- 第一版实现更适合主循环内短时阻塞读帧，不建议先塞进 ISR
- 软件结构应把“读帧”“活动判定”“灯控状态机”分开，不要混写
- 引脚命名建议使用协议化命名，例如 `TQ322_S_PIN`，不要继续沿用 `PIR_IN` 这类会误导实现的名字

## 维护边界

- 这里只保留 `TQ322` 器件级接口与时序事实，不记录具体板子的连线推断
- `7脚` 是否直连 `TQ322` 原始输出，仍应以原理图和项目侧真值文件为准
- 当前首版时钟建议只适用于已有参考实现讨论，不应在其他项目里直接当成既定配置
- 若后续新增更完整的引脚、电源或工作模式资料，应以原始器件资料复核后再补充
