# TQ322

ID: `components/tq322`

## Purpose

This summary supports device identification, interface modeling, and MCU-side read-frame implementation constraints for `TQ322`-related projects.

## Shared Conclusions

- `TQ322` must NOT be treated as a simple high/low-level `PIR` module by default
- The MCU side must understand the `S` pin of `TQ322` as a single-wire digital frame interface, not a simple level input
- Data is read starting from `MSB`
- Total frame length is `19bit`
- Frame format: `D18=1`, `D17=0`, `D16..D1=16bit signed data`, `D0=0`

## Critical Timing Constraints

- In passive or interrupt mode, after the `S` pin goes high for approximately `0.0625ms`, the MCU detects ready, then waits at least `100ns` before starting read timing
- In active or forced read mode, the MCU may actively pull the `S` pin high for at least `0.0625ms`
- For single-bit read, the MCU first pulls the `S` pin low with a low-level width of at least `200ns`
- Then generates a rising edge on the `S` pin and holds high for at least `200ns`
- After the sensor takes over, the MCU must sample at a fixed relative moment — each bit must not take a different instruction path
- After reading the `19th` bit, the MCU must force the `S` pin low and then immediately release it, or hold low for at least `64` system clock cycles, to trigger internal data latch update
- If aborting mid-read and wanting to update internal data, the `S` pin low level must exceed `1` system clock cycle; if forced high mid-way, reading terminates but data is not updated

## Direct MCU-Side Constraints

- Requires stable, controllable short-delay capability to meet `100ns` / `200ns`-level timing requirements
- The first implementation is better suited for short blocking reads in the main loop; do not force it into an ISR first
- Software structure should separate "read frame", "activity determination", and "light control state machine" — do not mix them
- Pin naming should use protocol-oriented naming, e.g. `TQ322_S_PIN`, do not continue using misleading names like `PIR_IN`
